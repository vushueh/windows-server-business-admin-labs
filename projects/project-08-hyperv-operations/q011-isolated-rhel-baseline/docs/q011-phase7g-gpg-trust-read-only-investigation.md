## Objective

Determine why the supervised RHEL 10.2 upgrade requested three Red Hat signing
keys even though the repositories reference the locally installed
`RPM-GPG-KEY-redhat-release` file. The window may compare the prompted public
fingerprints with Red Hat's published signing-key record, inspect the installed
key file and repository trust settings, list RPM's current public-key trust
state, and verify signatures on two already-cached RPM samples.

This is diagnosis only. It does **not** authorize a key import, package-cache
cleanup, repository refresh, DNF retry, package transaction, file change, or
network/service configuration change.

The guest trust-inspection commands are read-only. A future live approval must
separately and explicitly cover the temporary Hyper-V adapter/VLAN attachment,
VM start, normal guest shutdown, and final adapter isolation shown below.
Normal SSH/sudo authentication and service audit records may be generated by
execution; the window makes no managed configuration, trust, package, or cache
change.

Leonel later executed this run sheet under the exact Phase 7G approval. The
installed `redhat-release` package and key file verified, the two repositories

## Why The Investigation Is Separate

Phase 7 stopped fail-closed after all three unexpected import prompts were
answered `N`. DNF reported `upgrade_exit=1`, installed no key, and recorded no
new transaction. Q011 then returned to Off, disconnected, Untagged VLAN 0 with
`Phase7RecoveryPass=True`.

The public fingerprints displayed by DNF match Red Hat's published records:

| Key | Prompted fingerprint | Published use |
|---|---|---|
| Release key 2, `FD431D51` | `567E 347A D004 4ADE 55BA 8A5F 199E 2F91 FD43 1D51` | RHEL 6–10 |
| Auxiliary key 3, `5A6340B3` | `7E46 2425 8C40 6535 D56D 6F13 5054 E4A4 5A63 40B3` | RHEL 9–10 |
| Release key 4, `05707A62` | `FCD3 55B3 0570 7A62 DA14 3AB6 E422 397E 50FE 8467 A2A9 5343 D246 D627 6AFE DF8F` | RHEL 9.7 and 10.1 onward |

That public match is useful context, not authority to import anything. Red
Hat's [Product Signing Keys](https://access.redhat.com/security/team/key)
page is the authoritative fingerprint source. Red Hat also documents RHEL
10.1's RPM v6 / post-quantum signature behavior and read-only `rpmkeys -l` and
`rpmkeys -Kv` checks in
[Verifying RPM packages with post-quantum signatures](https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/9/html/security_hardening/verifying-rpm-packages-with-post-quantum-signatures_security-hardening).
The RHEL 10.2
[known issues](https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/10/html/10.2_release_notes/known-issues)
and [fixed issues](https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/10/html/10.2_release_notes/fixed-issues)
provide version-specific context for OpenPGP v6 and RPM multi-signature
handling. No diagnosis should be inferred from those notes until the local
read-only evidence is collected.

## Phase 7G-A — Fresh Host Preflight

Run in elevated Windows PowerShell on `WIN-PRQD8TJG04M`:

```powershell
& {
    $ErrorActionPreference = 'Stop'
    $VmName = 'Q011-RHEL102-BASELINE'
    $TargetSwitch = 'vSwitch-LAN'

    $Vm = Get-VM -Name $VmName
    $Adapters = @(Get-VMNetworkAdapter -VMName $VmName)
    $Vlan = Get-VMNetworkAdapterVlan -VMName $VmName
    $Dvd = @(Get-VMDvdDrive -VMName $VmName)
    $Snapshots = @(
        Get-VMSnapshot -VMName $VmName -ErrorAction SilentlyContinue
    )
    $Switch = Get-VMSwitch -Name $TargetSwitch
    $Confirmation = Read-Host (
        'After confirming the ASA is Off and no competing backup, storage, ' +
        'maintenance, Hyper-V, or Q011 work is active, type ' +
        'ASA-OFF-NO-COMPETING-WORK'
    )

    $Phase7GPreflightPass = (
        $env:COMPUTERNAME -eq 'WIN-PRQD8TJG04M' -and
        $Vm.State -eq 'Off' -and
        $Adapters.Count -eq 1 -and
        [string]::IsNullOrWhiteSpace($Adapters[0].SwitchName) -and
        $Vlan.OperationMode -eq 'Untagged' -and
        $Vlan.AccessVlanId -eq 0 -and
        $Dvd.Count -eq 1 -and
        [string]::IsNullOrWhiteSpace($Dvd[0].Path) -and
        $Snapshots.Count -eq 0 -and
        -not $Vm.AutomaticCheckpointsEnabled -and
        $Vm.AutomaticStartAction -eq 'Nothing' -and
        $Switch.SwitchType -eq 'External' -and
        $Confirmation -ceq 'ASA-OFF-NO-COMPETING-WORK'
    )

    [pscustomobject]@{
        ComputerName        = $env:COMPUTERNAME
        VmName              = $Vm.Name
        State               = $Vm.State
        Disconnected        = [string]::IsNullOrWhiteSpace(
            $Adapters[0].SwitchName
        )
        OperationMode       = $Vlan.OperationMode
        AccessVlanId        = $Vlan.AccessVlanId
        DvdEmpty            = [string]::IsNullOrWhiteSpace($Dvd[0].Path)
        SnapshotCount       = $Snapshots.Count
        TargetSwitch        = $Switch.Name
        Phase7GPreflightPass = $Phase7GPreflightPass
    } | Format-List

    if (-not $Phase7GPreflightPass) {
        throw 'Q011 Phase 7G preflight failed; do not attach or start the VM.'
    }
}
```

Stop unless `Phase7GPreflightPass=True`.

## Phase 7G-B — Restore Only The Proved Temporary Path

On the Hyper-V host, attach and start only Q011:

```powershell
& {
    $ErrorActionPreference = 'Stop'
    $VmName = 'Q011-RHEL102-BASELINE'
    $TargetSwitch = 'vSwitch-LAN'

    try {
        Set-VMNetworkAdapterVlan `
            -VMName $VmName `
            -Access `
            -VlanId 70 `
            -ErrorAction Stop

        Connect-VMNetworkAdapter `
            -VMName $VmName `
            -SwitchName $TargetSwitch `
            -ErrorAction Stop

        $Adapter = Get-VMNetworkAdapter -VMName $VmName
        $Vlan = Get-VMNetworkAdapterVlan -VMName $VmName
        $Phase7GAttachmentPass = (
            $Adapter.SwitchName -eq $TargetSwitch -and
            $Vlan.OperationMode -eq 'Access' -and
            $Vlan.AccessVlanId -eq 70

            VmName                 = $VmName
            SwitchName             = $Adapter.SwitchName
            OperationMode          = $Vlan.OperationMode
            AccessVlanId           = $Vlan.AccessVlanId
            Phase7GAttachmentPass  = $Phase7GAttachmentPass
        } | Format-List

        if (-not $Phase7GAttachmentPass) {
            throw 'Q011 Phase 7G attachment verification failed.'

        Disconnect-VMNetworkAdapter `
            -VMName $VmName `
            -ErrorAction SilentlyContinue

            -VMName $VmName `
            -Untagged `
            -ErrorAction SilentlyContinue
```

## Phase 7G-C — Inspect The Local Trust Inputs

Run these read-only commands through the proved SSH session. They must not
prompt for or import a key.

```bash
set -o pipefail

printf '%s\n' '=== release and RPM versions ==='
rpm -q redhat-release rpm

printf '%s\n' '=== package and key-file integrity ==='
rpm -qf /etc/pki/rpm-gpg/RPM-GPG-KEY-redhat-release
rpm -V redhat-release
redhat_release_verify_exit=$?
printf 'redhat_release_verify_exit=%s\n' "$redhat_release_verify_exit"
stat -c 'path=%n bytes=%s owner=%U group=%G mode=%a' \
  /etc/pki/rpm-gpg/RPM-GPG-KEY-redhat-release
sha256sum /etc/pki/rpm-gpg/RPM-GPG-KEY-redhat-release

printf '%s\n' '=== BaseOS and AppStream trust fields only ==='
sudo awk '
  /^\[/ {
    section=$0
    keep=($0=="[rhel-10-for-x86_64-baseos-rpms]" || $0=="[rhel-10-for-x86_64-appstream-rpms]")
  }
  keep && /^(enabled|gpgcheck|repo_gpgcheck|gpgkey)[[:space:]]*=/ {
    print section " " $0
  }
' /etc/yum.repos.d/redhat.repo

printf '%s\n' '=== RPM public-key trust state ==='
rpmkeys --list
if test -x /usr/lib/pqrpm/bin/rpmkeys; then
  /usr/lib/pqrpm/bin/rpmkeys --list
else
  printf '%s\n' 'pqrpm_rpmkeys_present=false'
fi
```

Interpret an empty `rpm -V redhat-release` output with exit `0` as package-file
verification passing. Any output or nonzero result is a stop condition, not
permission to reinstall the package.

## Phase 7G-D — Verify Two Already-Cached RPM Samples

The commands below search only the existing DNF cache. They do not refresh a
repository or clean the cache.

```bash
set -o pipefail

cache_roots=()
for root in /var/cache/dnf /var/cache/libdnf5; do
  test -d "$root" && cache_roots+=("$root")
done

test "${#cache_roots[@]}" -gt 0 || {
  printf '%s\n' 'cache_root_gate=false'
  exit 19
}

cache_count=$(sudo find "${cache_roots[@]}" -xdev -type f -name '*.rpm' | wc -l)
baseos_sample=$(sudo find "${cache_roots[@]}" -xdev -type f \
  -path '*rhel-10-for-x86_64-baseos-rpms*/*kernel-core-6.12.0-211.34.1.el10_2.x86_64.rpm' \
  -print -quit)
appstream_sample=$(sudo find "${cache_roots[@]}" -xdev -type f \
  -path '*rhel-10-for-x86_64-appstream-rpms*/*amd-gpu-firmware-20260609-23.el10_2.noarch.rpm' \
  -print -quit)

printf 'cached_rpm_count=%s\n' "$cache_count"
printf 'baseos_sample=%s\n' "$baseos_sample"
printf 'appstream_sample=%s\n' "$appstream_sample"

test -n "$baseos_sample" && test -n "$appstream_sample" || {
  printf '%s\n' 'cached_sample_gate=false'
  exit 20
}

sudo rpmkeys -Kv "$baseos_sample"
baseos_rpmkeys_exit=$?
sudo rpmkeys -Kv "$appstream_sample"
appstream_rpmkeys_exit=$?
printf 'baseos_rpmkeys_exit=%s\n' "$baseos_rpmkeys_exit"
printf 'appstream_rpmkeys_exit=%s\n' "$appstream_rpmkeys_exit"

if test -x /usr/lib/pqrpm/bin/rpmkeys; then
  sudo /usr/lib/pqrpm/bin/rpmkeys -Kv "$baseos_sample"
  baseos_pqrpm_exit=$?
  sudo /usr/lib/pqrpm/bin/rpmkeys -Kv "$appstream_sample"
  appstream_pqrpm_exit=$?
  printf 'baseos_pqrpm_exit=%s\n' "$baseos_pqrpm_exit"
  printf 'appstream_pqrpm_exit=%s\n' "$appstream_pqrpm_exit"
fi
```

Do not treat `NOKEY` or a trust-database absence as permission to import.
Invalid digest/signature output is a hard stop and must be reported exactly.
The two exact package names come from the retained Phase 7 transaction output;
the repository-specific cache paths prevent a sample from being attributed to
the wrong repository. Absence remains a safe stop, not authority to download.

## Phase 7G-E — Normal Shutdown And Final Isolation

After collecting only the approved output:

```bash
sudo systemctl poweroff
```

Wait up to three minutes for Q011 to become Off. Run on the Hyper-V host:

```powershell
$VmName = 'Q011-RHEL102-BASELINE'
$Deadline = (Get-Date).AddMinutes(3)
do {
    $Vm = Get-VM -Name $VmName
    if ($Vm.State -eq 'Off') {
        break
    }
    Start-Sleep -Seconds 5
} while ((Get-Date) -lt $Deadline)

if ((Get-VM -Name $VmName).State -ne 'Off') {
    Disconnect-VMNetworkAdapter -VMName $VmName
    Set-VMNetworkAdapterVlan -VMName $VmName -Untagged
    throw (
        'Q011 did not shut down within three minutes. Network containment ' +
        'was restored; forced power-off requires separate approval.'
    )
}

Disconnect-VMNetworkAdapter -VMName $VmName
Set-VMNetworkAdapterVlan -VMName $VmName -Untagged

$Vm = Get-VM -Name $VmName
$Adapters = @(Get-VMNetworkAdapter -VMName $VmName)
$Vlan = Get-VMNetworkAdapterVlan -VMName $VmName
$Dvd = @(Get-VMDvdDrive -VMName $VmName)
$Snapshots = @(
    Get-VMSnapshot -VMName $VmName -ErrorAction SilentlyContinue
)

$Phase7GEndStatePass = (
    $Vm.State -eq 'Off' -and
    $Adapters.Count -eq 1 -and
    [string]::IsNullOrWhiteSpace($Adapters[0].SwitchName) -and
    $Vlan.OperationMode -eq 'Untagged' -and
    $Vlan.AccessVlanId -eq 0 -and
    $Dvd.Count -eq 1 -and
    [string]::IsNullOrWhiteSpace($Dvd[0].Path) -and
    $Snapshots.Count -eq 0 -and
    -not $Vm.AutomaticCheckpointsEnabled -and
    $Vm.AutomaticStartAction -eq 'Nothing'
)

[pscustomobject]@{
    VmName               = $Vm.Name
    State                = $Vm.State
    AdapterCount         = $Adapters.Count
    Disconnected         = [string]::IsNullOrWhiteSpace(
        $Adapters[0].SwitchName
    )
    OperationMode        = $Vlan.OperationMode
    AccessVlanId         = $Vlan.AccessVlanId
    DvdEmpty             = [string]::IsNullOrWhiteSpace($Dvd[0].Path)
    SnapshotCount        = $Snapshots.Count
    Phase7GEndStatePass  = $Phase7GEndStatePass
} | Format-List
```

Require `Phase7GEndStatePass=True`.
